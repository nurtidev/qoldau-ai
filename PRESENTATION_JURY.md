# Qoldau AI — Презентация для жюри хакатона Astana Hub × Байтерек

> Demo Day · Astana Hub · 2026 · Russian language · solo-команда

---

## Слайд 1 — Что это

**Qoldau AI — единый портал поддержки бизнеса, где каждая мера господдержки задаётся как форма в no-code конструкторе, а не пишется в коде.**

Контекст: техническое задание хакатона требует реализовать услуги через конструктор форм. Иначе задание не зачитывается. Это и есть ядро проекта: один движок форм + один конструктор + одна база данных обслуживают все меры — от лизинга авиатранспорта в БРК-Лизинг до грантов молодёжного предпринимательства в Damu.

Аналог по подходу: Typeform / Tally — но для государственных мер поддержки, с расчётными полями, условной видимостью, mock eGov-префиллом и AI-генерацией формы из текстового описания.

Проверка no-code-природы за 10 секунд:

```bash
docker exec qoldau_postgres psql -U qoldau -d qoldau \
  -c "SELECT title, jsonb_array_length(form_schema->'steps') AS steps FROM services;"
```

Все шаги, поля, формулы, условия — в JSONB. Деплой при добавлении новой меры не нужен.

## Слайд 3 — Что реально работает

**Модуль 1. Конструктор форм (для администратора Байтерек / ведомства)**
- Создание услуги: название, описание, категория, орган-провайдер, **headline-параметры программы** (процентная ставка, максимальная сумма, срок)
- Визуальный редактор шагов и полей: text / textarea / number / currency / select / multiselect / date / file / checkbox / radio / calculated
- Расчётные поля с формулами (JS-выражения, переменные = `id` других полей)
- Условная видимость шагов и полей (operator: equals / not_equals / greater_than / less_than)
- Префилл из mock eGov по ИИН / БИН / названию организации
- Публикация / снятие с публикации — услуга появляется в каталоге без редеплоя

**Модуль 2. AI-ассистент для администратора**
- Кнопка «Сгенерировать форму через AI» в конструкторе
- Бэкенд-хендлер [backend/internal/handlers/ai.go](backend/internal/handlers/ai.go) (415 строк) шлёт описание в Anthropic Claude `claude-sonnet-4-6`
- Возвращается готовый `form_schema` JSON — администратор правит руками в визуальном конструкторе
- Снижает время создания новой меры с часов до минут

**Модуль 3. Личный кабинет предпринимателя**
- Вход по ИИН без пароля (имитация eGov), JWT-сессия
- Каталог услуг с фильтрами по категории, органу, поиску
- Подача заявки: пошаговая форма из `form_schema`, real-time пересчёт калькуляций, условные шаги
- Загрузка документов, история заявок, статусы, уведомления
- Eligibility Screener на главной: 4-шаговый онбординг (цель, отрасль, возраст бизнеса, выручка) → подбор подходящих программ со score-чипом и **параметрами программы** (ставка/сумма/срок)
- Виджет **«Перезвоните мне»** прямо в результатах онбординга — лид с именем и телефоном за 20 секунд (`POST /api/leads`)
- KGD-Check, Preflight-панель, AlternativeRecommendations — модули помощи заявителю

**Модуль 4. Админ-обработка заявок**
- Список всех заявок с фильтрами
- Изменение статуса (На рассмотрении → Одобрена / Отклонена / Доработка)
- Сводная аналитика: по услугам, статусам, органам, периодам

**Модуль 5. Калькулятор охвата + персонализированная рассылка** (сверх ТЗ — «до запуска программы»)
- В конструкторе формы — отдельный режим «Аудитория»: администратор настраивает фильтры (отрасль, регион, размер бизнеса, возраст бизнеса, выручка, возраст владельца, исключения по налоговой задолженности и реестру риска КГД) и в реальном времени видит число подходящих предпринимателей
- Разрезы аудитории: топ-5 по регионам, по отраслям, по размеру бизнеса — обновляются за ~300 мс при изменении фильтров
- Пример аудитории — 4 живые карточки реальных предпринимателей под текущие фильтры
- Кнопка «Отправить уведомление» → персонализированная рассылка с подстановкой `{{full_name}}` и `{{org_name}}` каждому получателю в личный кабинет
- Источник данных в MVP: 3000 синтетических предпринимателей в БД с распределением по 20 регионам РК, 8 отраслям, 4 категориям МСБ. В продакшне — eGov + КГД + ИС МСБ через ЕИШ

**Модуль 6. Воронка программы + анализ просадок** (сверх ТЗ — «после запуска программы»)
- В конструкторе формы — режим «Аналитика»: воронка от просмотров карточки до одобренной заявки с процентом просадки на каждом шаге
- Просадка с самым большим оттоком подсвечивается красным; клик по шагу → drilldown по полю, где заявители отвалились
- Drilldown показывает: статистики (p25 / медиана / p75) последних значений в брошенном поле + детерминированный insight по эвристикам
- Контрольный пример лизинга: 1434 просмотра → 250 начали → 162 шаг 2 → **44 шаг 3 (просадка 72%)** → 12 подано → 4 одобрено. Drilldown: «Запрашиваемая сумма лизинга», медиана 846 млн ₸, insight «лимит ниже реальной потребности целевой аудитории»
- **Замыкание контура с модулем 5**: кнопка «Перенастроить аудиторию под эту потребность» → автоматически открывает калькулятор охвата с предзаполненными фильтрами по выручке, соответствующей медиане
- Источник: ~17 000 синтетических просмотров + 1690 заявок + ~7000 шаговых событий за последние 60 дней. В продакшне — реальные события через хуки `POST /services/:id/view` и `POST /applications/:id/event`

**Модуль 7. Параметры программы + лид-захват «Перезвоните мне»**
- Услуги получили headline-параметры: процентная ставка, максимальная сумма, срок (миграция [008_program_terms.up.sql](backend/migrations/008_program_terms.up.sql) — реальные ставки/лимиты по 12+ программам Damu, БРК-Лизинг, КазАгроФинанс)
- Эти параметры показываются короткой полосой в карточках после онбординга: «ставка 6% · до 750 млн ₸ · срок до 84 мес.»
- В компоненте `EligibilityScreener` на главной странице — виджет «Перезвоните мне» с именем и телефоном (`POST /api/leads`, публично без авторизации)
- Лид сохраняется в новую таблицу `leads` со ссылкой на конкретную услугу, по которой обращение. Админ забирает список через `GET /api/leads`
- Бизнес-смысл: предприниматель, который видит сложную программу и не хочет сам заполнять форму, оставляет контакт за 20 секунд. Институту развития прилетает квалифицированный лид с привязкой к конкретной услуге

## Слайд 4 — Контрольный кейс «Лизинг» — глубина проработки

Сам факт работающей формы — это минимум. Жюри Байтерек ожидает реалистичности.

Контрольный кейс **«Приобретение авиатранспорта и вагонов в лизинг»** (АО «БРК-Лизинг») развёрнут в полную BRK-grade конфигурацию — 6 шагов, ~40 полей, миграция [003_realism_pack.up.sql](backend/migrations/003_realism_pack.up.sql):

1. **Информация о компании** — БИН, ОКЭД, дата регистрации, выручка за 3 года, численность
2. **Предмет лизинга и казсодержание** — авиатранспорт / вагоны / спецтехника, страна происхождения, доля каз. содержания, поставщик
3. **Финансовая модель** — аванс ≥ 15%, сумма лизинга, ставка, **DSCR** (целевой ≥ 1.3), **IRR** (≥ 12%), доля собственного участия (≥ 20%) — всё с автоматическим расчётом по формулам
4. **Обеспечение и оценка** — аккредитованный оценщик из реестра БРК, страховая компания
5. **KYC / ПОД-ФТ** — бенефициар > 25%, источник средств, согласия на ПКБ / ГКБ, ПДн (Закон РК № 94-V), санкционная декларация, согласие на запрос в КГД и ЕНПФ
6. **Документы** — фин. отчётность за 3 года, ТЭО, протокол участников, СТ-KZ при наличии

Это не просто «6 полей и кнопка Submit». Это структура заявки, которая знакома сотрудникам БРК-Лизинг по основной деятельности.

## Слайд 5 — Тиражируемость: одна архитектура → много мер поддержки

Помимо контрольного кейса, в базу засеяны меры от других институтов развития (миграция [002_seed_services.up.sql](backend/migrations/002_seed_services.up.sql) + 003):

| Орган | Меры |
|---|---|
| АО «БРК-Лизинг» | Лизинг авиатранспорта / вагонов (контрольный кейс) |
| Фонд «Damu» | Гарантирование кредитов МСБ (до 85% / 7 млрд ₸) · Субсидирование ставки ДКБ-2025 (до 6 п.п.) · Грант молодёжного предпринимательства (до 5 млн ₸) |
| «Демеу» | Льготное финансирование МСБ по ставке 6% |
| АО «КазАгроФинанс» / «АгроКапитал» | Льготное кредитование сельхозпроизводителей · Лизинг сельхозтехники |
| Экспортные программы | Страхование экспортных контрактов · Предэкспортное финансирование |
| Прочее | Грант для инновационных стартапов (Seed) и др. |

**Сейчас в БД:** ~16 предзаполненных услуг с реалистичной финансовой моделью каждая.
**Целевая ёмкость архитектуры:** 70+ мер господдержки, перечисленных в техническом задании.

Что важно для жюри: добавление новой меры — это SQL-INSERT в таблицу `services` либо нажатие кнопки в конструкторе. Не Pull Request, не релиз, не работа разработчика. Это и означает «нет потолка по числу услуг».

## Слайд 7 — Команда и статус

**Состав:** один разработчик, full-stack + интеграция AI. Backend (Go), frontend (React/TS), миграции, инфраструктура (Docker), интеграция Anthropic Claude — реализованы одним человеком.

**Стадия:** Working MVP. Система запускается командой `make up`, проходит сценарий жюри (создание услуги → AI-генерация формы → ручная правка → публикация → подача заявки пользователем → обработка администратором) за ~3 минуты.

**Тестовые входы для жюри:**
- Администратор: ИИН `000000000000`, любое имя
- Пользователь: любые 12 цифр в ИИН, любое имя
- URL: `http://localhost:3000` после `make up`

**Что подтверждено в коде и БД:**
- 16 услуг засеяно с реалистичными формами
- Контрольный кейс лизинга — 6 шагов, ~40 полей, DSCR/IRR/казсодержание/KYC
- AI-генерация формы из описания — рабочая, тестировалась
- Eligibility-движок — JSONB-правила, отдельная миграция

## Слайд 9 — Следующий шаг

У жюри три уровня участия — от лёгкого к серьёзному. Прошу выбрать один.

**1. Быстрая верификация (30 секунд, прямо сейчас).**
Назовите любую меру господдержки из списка 70+. Я создам её форму на сцене через AI-генерацию + ручную правку и опубликую за время разговора. Это снимает вопрос «работает ли no-code-движок на произвольной мере».

**2. Интро к куратору в Байтерек / Damu / БРК-Лизинг.**
Контакт человека, который курирует приём заявок по одной из мер. Цель — двухнедельный технический созвон, согласовать пилотный объём (одна мера, один регион) и формат интеграции.

**3. Победа в номинации хакатона + поддержка пилотной мобилизации.**
Призовое место даёт легитимность для входа в институт развития. Дальше — 60–90 дней до пилотного запуска одной меры на ограниченном контуре (одна категория получателей, один регион). Финансирование и доработку беру на себя.

Перспектива после пилота: пилот → регион → республиканское покрытие. Конкретные сроки этой части обсуждаются после результатов пилота, без обязательств жюри сейчас.
