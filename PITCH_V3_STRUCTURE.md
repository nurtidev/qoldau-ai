# 💾 Qoldau Score — данные группы и монетизация (материал слайда 11 и Q&A)

> Vision-материал. Со сцены — одной фразой (вшито в Блок 4). Названия дочек — по канону
> фактчека 10.07. ⚠️ Каждую строку про данные дочки перепроверить перед сценой так же строго,
> как майские программы: жюри — люди из этих организаций.

| Источник | Данные для профиля заёмщика |
|---|---|
| **БРК** | Крупные кредиты: обслуживание долга, реструктуризации, отраслевая концентрация |
| **ФРП** (бывш. БРК-Лизинг) | Промышленный лизинг: платёжная дисциплина, предметы лизинга, изъятия |
| **Даму** | Гарантии и субсидии МСБ: исполнение обязательств, дефолты по гарантиям |
| **АКК** | Агрокредиты: сезонность платежей, целевое использование |
| **КазАгроФинанс** | Агролизинг техники: дисциплина, залоговая история техники |
| **ЭКА (KazakhExport)** | Экспортные контракты, страховые случаи, валютная выручка |
| **Отбасы банк** | Накопительная дисциплина — поведенческий сигнал платёжеспособности |
| **КЖК** | Длинные жилищные обязательства, аренда с выкупом |
| **Портал Qoldau** | Заявки, черновики, дозапросы, поведение — копится с первого дня |
| **Госбазы (с согласия)** | КГД, ПКБ/ГКБ, данные eGov, ИСЖ |

**Уникальность:** кредитные бюро видят кредиты. Группа — единственная, кто видит **эффект господдержки**: что стало с бизнесом после меры. Таких данных нет ни у ПКБ, ни у банков.

**Три направления:** (1) скор-отчёты банкам по их слепым зонам — агро, село, стартовый МСБ (модель бюро: продаём оценку, не данные); (2) страховым — риск для агрострахования и страхования техники; (3) 🔑 **дизайн мер от данных**: кластеризация профилей → недоохваченные сегменты → мера, покрывающая наибольший % реальных заёмщиков, с симуляцией охвата до запуска (замыкается на живой экран «Калькулятор охвата»).

**Польза предпринимателю** (обязательно проговаривать при вопросе — критерий 9.9 требует ориентации на клиента, не на холдинг): профиль работает на заёмщика — быстрые предодобренные лимиты, меры, спроектированные под его сегмент.

**Юррамка** (при вопросе, самим не поднимать): модель кредитных бюро — скор с согласия субъекта, обучение на обезличенных данных, закон «О кредитных бюро…» и № 94-V о персданных.

# ❓ Q&A — 6 вероятных вопросов

1. **«У холдинга уже есть чат-бот — зачем ваш AI?»** *(троян из ТЗ п. 6.6 — «не дублировать существующий функционал»)* — «Мы его и не дублируем. Наш AI — не окно чата сбоку, он встроен в шаги пути: подбор меры в онбординге по структурированному профилю, проверка заявки на противоречия, инсайты автору услуги по воронке. Чат-бот отвечает на вопросы — наш AI выполняет шаги процесса.»
2. **«Госбазы — это реальные интеграции?»** — «Mock с обозначенной границей замены: контракт API повторяет реальные сервисы. Дайте доступ — подключим за спринт. То, что mock — интеграционная работа, не дефект продукта.»
3. **«Это скриншоты — а живьём работает?»** — «Работает, прямо сейчас: назовите любую меру с bgov.kz — соберу её форму через AI после питча, и пройдём путь заявителя живьём.»
4. **«Портал сам отказывает людям? А если модель ошиблась?»** — «Портал не принимает решений — это остаётся в BPM-системах дочек, как и требует ТЗ. Мы блокируем только заведомо непроходную подачу по формальным стоп-факторам, с причиной и альтернативой. Предоценка — справка по открытым данным, не решение о выдаче.»
5. **«Продавать скоринг банкам — законно? И отдадут ли дочки данные?»** — «Модель кредитного бюро: скор с согласия субъекта, обучение на обезличенных данных, продаём оценку — не данные. А дочкам не нужно ничего «отдавать»: холдинг — единый акционер, Qoldau — его инфраструктура. Начинаем с данных самого портала — они копятся с первого дня.»
6. **«Речевая аналитика записывает граждан — законно?»** — «Стандартное «разговор записывается» + обезличивание при обучении. Оцениваем сотрудника, не клиента. Это модуль дорожной карты — пилотируется на внутренних регламентах.»
